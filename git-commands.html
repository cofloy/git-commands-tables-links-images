50 Commonly used command line and what they do.

git config - to set the basic configurations on github like name and email.

git config --global user.name " " - This is used to set user name globally.

git config --global user.name - This is used to confirm that user name has been set.

git config --global user.email " " - This is used to set user email globally.

git config --global user.email - This is used to confirm that email has been set.

git config --global color.ui true - To see different colours on command line for different outputs.

git help - displays the main help documentation, showing a list of commonly used Git commands or to take help from github help section for different commands and other errors.

git init - create an empty Git repository or reinitialize an existing one.

git add . or git add --all - To add all files in the current folder

git commit -m "message" or git commit -message "message" - creates a new commit with the changes in the staging area and specifies the commit message inline or to commit our changes(taking a snapshot) and providing a message to remember for future reference.

git commit -a -m - To add any of our tracked files to staging area and commit them by providing a message to remember.

git commit -a or git commit --all - commits all modified and deleted files in the repository without explicitly using git add to stage the changes.

git reset -soft HEAD^ - to undo last commit and bring file to staging area.

git reset -hard HEAD^ - to undo last commit and remove file from the staging area as well(In case we went horribly wrong).

git reset -hard HEAD^^ - to undo last 2 commits and all changes.

git remove rm - to remove remote from our local repository

git branch - lists all branches in the repository and current branch that we are working on.

git branch -M -creates a new branch which is a new version of the repository as it appears when added, -M helps to move the name to the main

git branch branch-name - creates a new branch with a specified name

git branch -d branch-name- deletes the specified branch

git branch -D -to forcefully delete a branch without making commits

git branch -a -lists all local and remote branches

git branch -r -lists all remote branches

git merge branch-name - merges the specified branch into the current

git log - displays the commit history of the current branch

git remote add origin - connnects the local folder to the repository on Github. It is followed by the repository's link.

git remote add adress -to add new remotes to our local repository for a particular git address.

git push -u origin main - pushes code to Github. The -u flag creates a tracking reference for the branch and origin main puts the code in the main branch.

git clone - used to make a copy of a repository from an existing url

git status - shows the current state of a repository including tracked and untracked files, modifiled files and branch information

git status -ignored - displays ignored files in addition to the regular status output

git diff - shows the changes between the working directory and the staged area(index)

git diff commit1 commit2 - displays the differences between two commits

git diff -staged or git diff -cached - displays the changees between the staging area(index) and the last commit

git diff HEAD - displays the difference between the current directory and the last commit

git restore file - restores a file in the working directory to its state in the last commit

git reset commit - moves the branch pointer to a specified commit, resetting the staging area and the working directory to match the specified commit.

git rm file - removes a file from both the working directory and the repository, staging the deletion.

git mv - moves or renames a file or directory in your Git repository

ls - to see directories and files in the current directory.

ls -la - to see hidden directories and files within the current directory.

mkdir -to make a directory that is, to create a folder if not created initially

touch - to create a single file

cd - to change directory that is, to move from one folder to another

pwd - present working directory that is, to display the name pf the current folder

git switch main - to switch back to the main branch

git tag - to see the list of available tags.

git fetch - to fetch down any changes from global repository to current repository

git stash - to move staged files to stash area which are present in staging area.

git rebase - performs three tasks which are ; firstly, to move all changes to main which are not in origin/main to a temporary area. secondly, run all origin main commits. thirdly, run all commits in the temporary area on top of our main one at a time, so it avoids merge commits
